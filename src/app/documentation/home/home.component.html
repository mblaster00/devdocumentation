Introduction

The purpose of this site is to detail the design principles, objects, behaviours and error handling for the Logidoo Ecommerce API. The overriding goal of the API is to enable all parties to implement Logidoo Ecommerce.

Principles

APIs in a flexible, yet consistent manner. We hope to achieve this by the implementing the following principles:

  • Use of REST architectural principles.
  • Use of ISO international standards for enumerators such as currency and country codes

Overview

  1. You need a valid API key to send requests the API endpoints. You can get key from the integrations dashboard.
  2. The API has an access rate limit applied to it.
  3. The Ecommerce API will only response to secured communication done over HTTPs. HTTP requests will be sent a 301 redirect to corresponding HTTPs ressources.
  4. Response to every request is sent in JSON format . In case the API request results in an error, it is represented by an "Error": {{ '{' }} }
  5. The request method (verb) determines the nature of action you intend to perform. A request made using the GET method implies that you want to fetch something from Logidoo, and POST implies you want to save something new to Logidoo.
  6. The API call will respond with appropriate HTTP status code for all requests. Within Ecommerce API when a response is received, the status code is highlighted and accompanied by a help text that indicates the possible meaning of the response code. A 200 Ok indicates that all went well, while 4XX or 5XX response codes indicate an error from the requesting client or our API servers respectively.

Authentication

  • An API Key is required to be sent as part of every request to the Ecommerce API, in the form of an secret-token request header.
  • If you do not have an API Key, you can easily generate one by heading over to the Ecommerce Integrations Dashboard.
  • An API Key tells our API server that the request it received came from you. Everything that you have access to in Ecommerce is accessible with an API Key that is generated by you.
  • For ease of use inside Ecommerce, you could store your API key in an environment variable called SECRET_KEY and this Collection will automatically use it to make API calls.
